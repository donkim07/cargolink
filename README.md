## 🌍 The Problem We're Solving

</div>

<br/>

Africa's **$180B+ freight logistics sector** is broken in five predictable ways — and every existing solution ignores at least three of them.

<br/>

<div align="center">

| &nbsp; | Pain Point | The Reality |
|:---:|---|---|
| 📱 | **Internet dependency** | 40%+ of Tanzania has no smartphone — yet cargo must move |
| 💸 | **Opaque pricing** | Shippers overpay by **30–50%** due to information asymmetry |
| 🚚 | **Empty trucks** | Vehicles run **60%+ empty** on return journeys — pure waste |
| 💳 | **Cash-only markets** | Mobile money is ubiquitous, but freight platforms ignore it |
| 🗺️ | **No route intelligence** | Drivers navigate floods, wildlife zones, and roadworks blind |

</div>

<br/>

## ✨ Features

<br/>

<table>
<tr>
<td width="50%">

### 🔐 Africa-First Authentication
OTP-only login via Africa's Talking SMS. No passwords. No Google OAuth. Works on **any phone** — including feature phones via USSD. JWT access + refresh tokens issued post-verification. Redis-backed rate limiting baked in.

</td>
<td width="50%">

### 📦 Dynamic Freight Marketplace
Post cargo with type, weight, volume, urgency, and refrigeration needs. A real-time **pricing engine** calculates cost instantly: base rate × distance + urgency multiplier + weight surcharge + insurance. Google Maps Distance Matrix for auto ETA.

</td>
</tr>
<tr>
<td width="50%">

### 🔨 Live Auction System
Customers post shipments to open auction. Providers compete with real-time bids + countdown timers. SMS fires on every bid. One-click winner selection auto-creates a confirmed booking.

</td>
<td width="50%">

### 🤝 Shared Cargo / LTL Marketplace
Providers publish spare truck capacity on fixed routes. Multiple shippers split a single vehicle. Visual capacity bar tracks remaining tonnage live. **Directly solves the 60% empty truck problem.**

</td>
</tr>
<tr>
<td width="50%">

### 📡 USSD Tracking — Zero Internet
```
*384*123#
├── 1. Track Shipment
├── 2. Delivery Status
├── 3. Payment Status
└── 4. Contact Support
```
Any phone. Any network. Full cargo visibility — offline.

</td>
<td width="50%">

### 🗺️ Route Intelligence Engine
Corridor risk analysis for Tanzania's major freight routes — flood alerts (Kilosa zone), wildlife crossings (Mikumi), road works (Shinyanga → Mwanza), weather (Moshi highlands). Community hazard reporting via Redis pub/sub.

</td>
</tr>

## 🚀 Quick Start

<br/>

**Prerequisites:** Python 3.11+ · Node.js 20+ · PostgreSQL 15+ · Redis 7+

<br/>

### Backend

```bash
git clone https://github.com/your-username/cargolink-africa.git
cd cargolink-africa/backend

python -m venv venv && source venv/bin/activate
pip install -r requirements.txt

cp .env.example .env          # Add your API keys

alembic upgrade head          # Run migrations
uvicorn app.main:app --reload --port 8000
```

> 💡 Set `DEBUG=true` to auto-seed demo providers, vehicles, shipments, and auctions on startup.

<br/>

### Frontend

```bash
cd ../frontend
npm install
cp .env.example .env.local    # Add VITE_API_BASE_URL + VITE_GOOGLE_MAPS_API_KEY
npm run dev
```

Open **http://localhost:5173** — platform is live. API docs at **http://localhost:8000/docs**.

|:---:|---|---|

## 💰 Pricing Engine

<br/>

```python
# services/pricing.py
BASE_RATES      = { "motorcycle": 200, "van": 350, "pickup": 400,
                    "truck": 500, "refrigerated_truck": 700, "container": 900 }  # TZS/km

URGENCY         = { "standard": 1.0, "express": 1.3, "urgent": 1.6 }

# ──────────────────────────────────────────────────────────────────
#  total = (base_rate × distance × urgency)
#        + (weight_tons × 5,000)          ← weight surcharge
#        + (50,000 if refrigerated)        ← cold chain fee
#        + 8% platform fee
#        + 2% insurance
# ──────────────────────────────────────────────────────────────────
```

All values in **TZS (Tanzanian Shillings)** — formatted as `TZS 450,000`.
